A.3 — Spacing, Layout & Grid

Base Spacing Unit

Base unit: 8px. All spacing values are multiples of 8.

XS: 8px · SM: 16px · MD: 24px · LG: 32px · XL: 48px · 2XL: 64px · 3XL: 96px

Section bottom margin: 96px. Card padding: 28px 32px. Page padding: 88px 52px.

Max Content Width

Web (Admin / Customer menu): max-width: 1140px, centered.

Admin sidebar layout: 240px fixed sidebar + fluid content area.

Waiter App: Full-width native. No max-width constraint. Optimized for ~360px–430px Android screens.

Grid System

2-column: 1fr 1fr · 16px gap
3-column: 1fr 1fr 1fr · 16px gap
4-column: 1fr 1fr 1fr 1fr · 12px gap

Mobile breakpoints: Stack to 1-column below 900px for 2/3/4 col grids; below 720px for all others.

A.4 — Shared Component Library

Buttons

  • Primary: bg #FF4D00, text #fff, font Space Mono, 13px, uppercase, letter-spacing 2px, padding 16px 32px, no border-radius (sharp corners everywhere)
  • Secondary: bg transparent, border 2px solid #FF4D00, text #FF4D00
  • Destructive: bg transparent, border 2px solid #FF4D00, hover fills red
  • Ghost: no border, text #666, hover text #E8E8E8
  • Loading state: Spinner replaces label. Disabled opacity 0.4.

Input Fields

  • bg: #1A1A1A, border: 1px solid #2A2A2A, focus: border-color #FF4D00
  • Padding: 12px 16px. Font: Space Grotesk 15px. Placeholder: #666
  • Error state: border-color #FF4D00, red helper text below
  • No border-radius on any input. All corners square.
  • Search inputs get magnifier icon left-inset

Cards & Surfaces

  • Base card: bg #111, border 2px solid #2A2A2A, padding 28px 32px
  • Accent cards: 6px solid box-shadow in accent colour (red/yellow/white/green)
  • Hover state: background lifts to #1A1A1A
  • Modal overlay: bg rgba(0,0,0,0.85), card centered with accent border-top
  • Toasts: right-side slide-in, accent left-border, 3s auto-dismiss

Tags, Badges & Pills

  • Badge: bg #1A1A1A, 1px border, Space Mono 10px, 4px 10px padding
  • Colour variants: Red (--accent) / Yellow (--accent2) / Green / White
  • Status pills: AVAILABLE (green), OCCUPIED (yellow), BILL REQUESTED (red), SETTLED (muted)
  • Veg dot: solid green circle 10px. Non-veg: solid orange/red square 10px.
  • Discount badge: bg --accent2, text #000, overlaid on item card top-left

B.1 — Theme & Visual Identity

Base Theme

  • Dark base (#000) — matches global DMX design system
  • Per-restaurant accent colour override (admin-configurable)
  • Restaurant logo displayed in app header, top-center
  • Restaurant name in Space Grotesk 700, 24px below logo

UI Personality

  • Large, tap-friendly touch targets — min 48×48px all interactive elements
  • Menu items: image-led cards, max 2-column grid on mobile

Waiter App

React Native (Expo) Android app installed on the waiter's phone. Receives push notifications for new orders and bill requests. Provides a live floor view and order management interface. Designed for extreme speed in a fast-paced environment.
React NativeExpoAndroidFCM PushOffline Queue
C.1 — Theme & Visual Identity

Theme

  • Same global DMX dark theme — #000 base, #FF4D00 accent
  • Large, bold typography — minimum 16px body, 24px+ for critical info (table numbers, totals)
  • High contrast for readability in dim restaurant lighting and movement
  • Touch targets minimum 56px height for fast fat-finger use
  • Minimal animations — speed over polish. No decorative motion.

Status Colour System

Every table on the floor is represented by a coloured status indicator:

Free Occupied Bill Requested Settled

Bill Requested pulses / vibrates phone. Tables waiting >20 min flagged with warning icon.

E.1 — Theme & Design Philosophy

Visual Identity

  • Per-restaurant custom design — not a template. Designed to match the restaurant's ambiance (upscale, casual, fast-casual).
  • 3 design tiers: Classic (elegant serif + warm tones), Bold (similar to DMX design system — dark + accent), Natural (earthy, light mode)
  • Restaurant owner selects preferred aesthetic during onboarding
  • Restaurant logo, colours, and photography are core design inputs
  • DMX branding NOT visible on the restaurant site — fully white-label

Technical Standards

  • Core Web Vitals: LCP <1.5s, CLS <0.1, FID <100ms (Google ranking factors)
  • Mobile-first responsive (Google mobile-first indexing)
  • Next.js SSG/ISR for near-instant page loads and SEO crawlability
  • Restaurant schema markup (LocalBusiness, Restaurant, Menu, Review schemas)
  • Open Graph + Twitter Card metadata on all pages
  • Sitemap.xml + robots.txt auto-generated

Cross-App Navigation & Data Flow
How every surface in the DMX ecosystem connects — triggers, real-time events, and data relationships between the four applications.

Real-Time Event Flow (Socket.io)

  • Customer places order → Server emits new_order event → Waiter app push + Admin dashboard live feed updates simultaneously
  • Customer taps "Call Waiter" → Server emits bill_request → Waiter push (with total) + Admin overview pending bills
  • Waiter marks settled → Server emits table_settled → Customer app shows "Thank You", Admin dashboard updates revenue counter + analytics, table becomes free on floor view
  • Admin toggles item unavailable → Server emits menu_update → Customer app live menu reflects change within seconds
  • Admin pushes discount → Server emits discount_active → Customer app banner appears immediately

Cross-App Shared Data

  • Restaurant config (name, logo, theme) → Admin Settings → Customer App (branding), Waiter App (restaurant name), Website (fetched via API)
  • Menu data → Admin Menu Management → Customer App (live menu), Website (E-03, ISR cached)
  • Table data → Admin Settings → Customer App (QR URLs), Waiter App (floor map)
  • Order data → Customer App → Waiter App (push) → Admin (analytics feed)
  • Profit scores → AI Engine (weekly) → Admin Dashboard (D-06) → Menu Management (D-03 badges)
